Product Architecture

How data flows, how logic is applied, and where boundaries exist

Design philosophy

LumiLedger is architected around a core principle: data preparation should be transparent, traceable, and grounded in accounting logic.

The system does not attempt to replace professional judgment. It prepares financial data in a way that makes professional judgment easier to apply.

Every design decision reflects this boundary.

Source-driven data model

LumiLedger is built on a source-driven architecture. Every transaction must be tied to a known, connected financial account.

This approach ensures that coverage is explicit. If an account is not connected, its activity is not included. This prevents silent incompleteness.

Users always know what is being measured and what is not.

Data flow

Account connection

Business owners connect bank accounts, credit cards, and loans through secure integrations. These become the system's known sources of financial activity.

Transaction ingestion

Transactions are imported daily from connected accounts. Each transaction includes date, amount, description, and source account information.

Journal entry generation

For each transaction, a journal entry is created. One side is anchored to the known source account. The offsetting side is classified using machine-learned logic and AI assistance.

Financial statement construction

Journal entries are aggregated into financial statements following standard accounting structure. Income Statements, Balance Sheets, and General Ledger views are generated automatically.

Narrative generation

KPIs are calculated from the financial statements. A plain-English narrative is generated to explain trends, changes, and notable patterns in performance.

Classification logic

LumiLedger classifies transactions using a combination of machine-learned patterns and AI-assisted reasoning.

Each classification is transparent and includes a confidence score. Users and advisors can review, adjust, or override any classification.

Over time, the system learns from corrections and adapts to business-specific patterns.

Handling commingled accounts

LumiLedger recognizes that many small businesses use personal accounts for business activity.

Personal accounts can be connected intentionally. Each transaction is classified as either business or personal at the transaction level.

This allows for accurate separation while maintaining complete visibility into all activity.

Balance sheet integrity

LumiLedger maintains balance sheet integrity by tracking all connected account balances and ensuring that journal entries always balance.

Cash and credit card liabilities are reconciled against known account balances. Discrepancies are surfaced rather than hidden.

This prevents drift between reported financials and actual account positions.

Data boundaries

LumiLedger does not infer activity from accounts that are not connected. It does not estimate missing data or fill gaps with assumptions.

Coverage is explicit. Completeness is visible.

This boundary is intentional and reflects a commitment to transparency over false confidence.

System boundaries

LumiLedger does not make tax elections, enforce accounting method choices, or assert compliance authority.

It generates financial statements for understanding and analysis, not for filing or formal compliance.

When compliance is required, LumiLedger hands off clean, structured data to an accounting platform where professional judgment is applied.

This boundary preserves professional authority and prevents the system from overstepping its role.

Export and integration

LumiLedger supports clean export of journal entries, financial statements, and transaction detail.

Data can be transferred to QuickBooks, Excel, or other accounting platforms when formal record-keeping is required.

This flexibility allows LumiLedger to serve as a preparation layer without locking users into a single ecosystem.

Security and privacy

All financial data is encrypted in transit and at rest. Account connections use industry-standard secure integrations.

Data is never sold or used for purposes outside of the user's financial preparation.

Access controls ensure that only authorized users and advisors can view financial information.

Scalability and performance

LumiLedger is built to handle businesses with hundreds of transactions per month across multiple accounts.

The system processes transactions in near real-time and updates financial statements daily.

Performance remains consistent as transaction volume grows, ensuring that the system can support businesses as they scale.

Continuous improvement

Classification logic improves over time as more data is processed and more corrections are applied.

System updates are deployed regularly to improve accuracy, add features, and refine the user experience.

Feedback from users and accounting professionals is incorporated into ongoing development.
